# examples/README.md

本目录放的是**示例**，不是项目事实。

作用：

- 帮助开发者快速理解各文档“填得好”的样子
- 帮助理解输出风格和颗粒度
- 为项目落地文档提供可复制的参考

当前示例包括：

- `EXAMPLE_FILLED_TECH_SOLUTION.md`
  - 一个已填充的前端技术方案示例
- `EXAMPLE_API_CONTRACT_ENTRY.md`
  - 一个接口契约条目的示例
- `EXAMPLE_TASK_BREAKDOWN_ADD_LIST_PAGE.md`
  - 一个新增列表页任务拆解示例
- `EXAMPLE_BUSINESS_GLOSSARY.md`
  - 一个业务术语和状态说明示例
- `react/EXAMPLE_COMPONENT_PATTERN_REACT.md`
  - React + TSX 风格的组件模式示例
- `vue/EXAMPLE_COMPONENT_PATTERN_VUE.md`
  - Vue 3 + SFC 风格的组件模式示例

使用建议：

1. 复制最近似的示例
2. 替换为项目真实术语、目录、接口、角色、页面
3. 删除不适用内容
4. 只保留与当前项目技术栈一致的示例
5. 不要把示例中的业务内容误当作你项目的真实规则

落地到真实项目时，如果项目只使用 React 或只使用 Vue，建议删除另一种技术栈示例，避免 AI 误用。
