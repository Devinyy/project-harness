## 1. 术语示例

- 商家：平台中的合作主体，可拥有多个门店和多个活动实例
- 门店：商家下的实际经营单元，通常与地理位置相关
- 活动：面向用户投放的一次运营策略实例
- 模板：用于快速创建活动的结构定义，本身不直接生效
- 实例：基于模板创建的真实活动对象，可进入生效周期

## 2. 易混淆概念

- 商家 != 门店
- 模板 != 活动实例
- 发布 != 生效
- 停用 != 删除

## 3. 角色示例

### 运营

- 可创建和编辑活动
- 可查看商家列表
- 不能修改权限配置

### 管理员

- 可访问全部模块
- 可分配角色
- 可查看全量数据范围

## 4. 活动状态机示例

- 状态集合：草稿、待审核、审核通过、审核驳回、生效中、已结束、已停用
- 初始状态：草稿
- 流转路径：
  - 草稿 -> 待审核
  - 待审核 -> 审核通过 / 审核驳回
  - 审核通过 -> 生效中
  - 生效中 -> 已结束 / 已停用
- 规则：
  - 已结束不可编辑
  - 已停用不可再次编辑，只可复制为新活动
  - 审核驳回可返回草稿修改后再次提交
