## 执行步骤

### Step 1: 生成自测清单

从以下来源提取自测点：

1. **技术方案**中的"验收标准"和"自测清单"
2. **Playbook**中对应任务类型的"容易漏掉的点"
3. **PRD**中的功能要求和异常场景

整理为结构化自测清单：

```
## 自测清单

### 正常流程
- [ ] 测试点 1：描述
- [ ] 测试点 2：描述

### 边界场景
- [ ] 空数据展示
- [ ] 加载状态
- [ ] 接口错误
- [ ] 权限不足

### 角色差异
- [ ] 角色 A 可执行操作 X
- [ ] 角色 B 不可见按钮 Y

### 交互细节
- [ ] 提交防重复
- [ ] 表单校验
- [ ] 成功后刷新 / 跳转
- [ ] 返回列表保留上下文
```

### Step 2: 编写 Midscene 测试脚本

根据自测清单，使用 Midscene.js 编写自动化测试：

```typescript
// 示例结构，具体 API 以 midscenejs.com 文档为准

import { test } from '@midscene/web';

test('商家列表页 - 正常加载', async ({ ai, aiAssert }) => {
  // 导航到目标页面
  await ai('打开商家列表页');

  // 验证页面元素
  await aiAssert('页面标题显示为"商家管理"');
  await aiAssert('筛选栏包含关键词搜索、状态筛选');
  await aiAssert('表格显示商家数据，至少包含名称、状态、操作列');
});

test('商家列表页 - 筛选功能', async ({ ai, aiAssert }) => {
  await ai('打开商家列表页');
  await ai('在状态筛选中选择"已停用"');
  await ai('点击搜索');
  await aiAssert('表格中所有商家状态显示为"已停用"');
});

test('商家列表页 - 空态', async ({ ai, aiAssert }) => {
  await ai('打开商家列表页');
  await ai('在搜索框输入一个不存在的商家名称');
  await ai('点击搜索');
  await aiAssert('显示空数据提示');
});
```

### Step 3: 执行测试

1. 启动本地开发环境
2. 运行 Midscene 测试脚本
3. 收集测试结果和截图

### Step 4: 生成自测报告

```markdown
# 前端自测报告

## 基本信息
- 需求名称：
- 测试日期：
- 测试环境：
- 测试账号/角色：

## 测试结果总览
- 总用例数：
- 通过：
- 失败：
- 跳过：

## 详细结果

### 正常流程
| 测试点 | 结果 | 备注 |
|--------|------|------|
| 页面正常加载 | ✅ | — |
| 筛选功能正常 | ✅ | — |
| 分页正常 | ✅ | — |

### 边界场景
| 测试点 | 结果 | 备注 |
|--------|------|------|
| 空数据展示 | ✅ | — |
| 接口错误展示 | ⚠️ | 错误提示文案待确认 |
| 权限不足 | ✅ | — |

### 发现的问题
1. 问题描述 + 复现步骤 + 截图
2. ...

### 遗留项
- 待后端联调验证的点
- 待不同角色验证的点
- 需要更新的文档
```

### Step 5: 问题处理

- 测试失败 → 参考 `docs/12_TROUBLESHOOTING.md` 排查
- 排查后修复代码 → 重新执行相关测试
- 确认为非前端问题 → 记录到报告的遗留项

---

## 输出

- Midscene 测试脚本文件
- 自测报告（Markdown）
- 测试截图（如有）

---

## 质量标准

- 自测清单覆盖技术方案中的所有验收标准
- 正常流程、边界场景、角色差异三个维度均有用例
- 每个失败用例有明确的复现步骤
- 遗留项有明确的负责人和跟进计划
- 自测通过后才可提交代码
